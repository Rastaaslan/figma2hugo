6. Niveaux de titre

Pour forcer un vrai niveau HTML sans ambiguite

titre-h1-* titre-h2-* titre-h3-* jusqu'a h6

  hero-content
    titre-h1-prestation
    texte-hero

section-services
  section-block-services
    titre-h2-services
    card-service-1
      titre-h3-service-audit
      texte-service-audit
    card-service-2
      titre-h4-service-rgpd
      texte-service-rgpd
Convention recommandee Utiliser titre-h1-* a titre-h6-*. Le moteur comprend aussi heading-h2-* si besoin.
Fallback existant Si aucun niveau n'est precise, le moteur garde son heuristique actuelle selon le role et la taille du texte.

7. Conventions qui aident vraiment le responsive

Ce que le moteur lit mieux, et ce qui vous evitera des blocs figes

Utiliser des Frame pour les conteneurs semantiques Reserve les Group au decoratif. Un vrai bloc web doit vivre dans un conteneur clair.
Ajouter de l'Auto Layout partout ou le contenu est en flux Piles verticales, lignes de cards, colonnes de formulaire, lignes contact : tout cela gagne a etre structure comme du flux plutot que comme une photo de maquette.
Separer contenu, media et decor Typiquement content-*, media-*, decor-*, bg-*. Plus ces intentions sont separees, plus le passage a une mise en page fluide est possible.
Nommer explicitement les lignes et colonnes Utiliser row-*, col-* ou ligne-* aide a faire comprendre l'intention de layout.
Eviter les wrappers anonymes Un wrapper sans nom metier est souvent le premier endroit ou le moteur perd l'intention de layout.
Le vrai objectif n'est pas seulement d'avoir un naming propre. C'est d'aider le pipeline a distinguer ce qui doit rester decoratif et absolu, de ce qui peut devenir une section web en flux, puis plus tard une section vraiment responsive.

Raccourci conventions

  • section-* : grande section de page
  • section-block-* : sous-bloc coherent
  • titre-h1-* a titre-h6-* : niveau de titre explicite
  • row-*, col-*, ligne-* : layout
  • bg-* : fond
  • decor-* : decor
  • image-* : media
  • titre-*, texte-*, libelle-* : texte
  • formulaire-*, input-*, button-* : formulaires
  • link-grid-*, href-card-* : cards
  • accordion-* : FAQ / disclosure
  • carousel-* : carousel

A eviter

  • contact-formulaire au lieu de formulaire-contact
  • input-ligne-contact pour un simple wrapper de layout
  • recycler titre-* pour les labels de link cards
  • melanger fond, contenu et decor dans un seul frame
  • laisser des groupes intermediaires sans nom metier
